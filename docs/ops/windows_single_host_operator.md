# Windows Single-Host Operator Guide

This document is the authoritative operator handoff for the supported Windows-first deployment target. It covers fresh install, upgrade, backup, restore, rollback, and secret rotation for the supported runtime surface:

- the FastAPI facade at `service.api_server`
- the installed `earctl` CLI for administrative tasks that do not depend on a source checkout

Use this guide for deployed hosts. `README.md`, `RUNBOOK.md`, `service/windows/*.md`, and `docs/api/readme.md` should defer to this file for lifecycle operations.

## Scope and support boundary

- Supported target: one Windows host running one EarCrawler API service.
- Supported deployment artifact for the API service: the signed/published wheel installed into a dedicated Python virtual environment.
- Supported service wrapper: NSSM (`nssm.exe`) pointing at that virtual environment.
- Rate limits, concurrency controls, and the RAG cache are process-local. This guide does not claim multi-instance correctness.
- Not the operator deployment path: a source checkout, `earctl api start`, `scripts/api-*.ps1`, the PyInstaller `earctl.exe`, or the Inno Setup installer. Those are repo-local or convenience surfaces, not the authoritative API hosting path.
- This guide assumes the configured Fuseki SPARQL query endpoint already exists. EarCrawler does not currently automate Fuseki provisioning.

For the deferred future-work note on scale-out design, see
`docs/ops/multi_instance_deferred.md`.

## Standard host layout

| Path | Purpose |
| --- | --- |
| `C:\Program Files\EarCrawler\runtime` | Dedicated runtime root |
| `C:\Program Files\EarCrawler\runtime\.venv` | Python virtual environment that hosts the wheel |
| `C:\ProgramData\EarCrawler\workspace` | Working directory for reports and runtime artifacts |
| `C:\ProgramData\EarCrawler\config` | Deployment-owned configuration files |
| `C:\ProgramData\EarCrawler\logs` | Service stdout/stderr logs |
| `C:\ProgramData\EarCrawler\backups` | Operator-created backup snapshots |

## Required inputs

Before touching the host, collect:

- the signed release wheel, for example `earcrawler-0.2.5-py3-none-any.whl`
- the release checksum file generated by `scripts/checksums.ps1`
- Python 3.11 on the host
- NSSM installed on the host
- the Fuseki SPARQL query endpoint URL, for example `http://127.0.0.1:3030/ear/query`
- any required secrets:
  - `EARCRAWLER_API_KEYS` for inbound API auth if you are not running anonymously
  - `TRADEGOV_API_KEY` for scheduled ingest jobs
  - `GROQ_API_KEY` or `NVIDIA_NIM_API_KEY` only if optional remote answering is enabled

## Fresh install

### 1. Prepare directories

Run PowerShell as Administrator:

```powershell
$runtimeRoot = 'C:\Program Files\EarCrawler\runtime'
$workspaceRoot = 'C:\ProgramData\EarCrawler\workspace'
$configRoot = 'C:\ProgramData\EarCrawler\config'
$logRoot = 'C:\ProgramData\EarCrawler\logs'
$backupRoot = 'C:\ProgramData\EarCrawler\backups'

New-Item -ItemType Directory -Force -Path $runtimeRoot, $workspaceRoot, $configRoot, $logRoot, $backupRoot | Out-Null
New-Item -ItemType Directory -Force -Path (Join-Path $workspaceRoot 'kg\reports') | Out-Null
New-Item -ItemType Directory -Force -Path (Join-Path $workspaceRoot 'run\logs') | Out-Null
```

### 2. Install the release wheel into a dedicated virtual environment

```powershell
$runtimeRoot = 'C:\Program Files\EarCrawler\runtime'
$venvPython = Join-Path $runtimeRoot '.venv\Scripts\python.exe'

py -3.11 -m venv (Join-Path $runtimeRoot '.venv')
& $venvPython -m pip install --upgrade pip
& $venvPython -m pip install 'C:\path\to\earcrawler-0.2.5-py3-none-any.whl'
& $venvPython -m pip show earCrawler
```

### 3. Set machine-scoped runtime configuration

The supported operator path uses machine environment variables so the service can be rebuilt without depending on per-user profiles or a source checkout.

```powershell
[System.Environment]::SetEnvironmentVariable('EAR_SYSTEM_INSTALL', '1', 'Machine')
[System.Environment]::SetEnvironmentVariable('EAR_TELEMETRY_CONFIG', 'C:\ProgramData\EarCrawler\config\telemetry.json', 'Machine')
[System.Environment]::SetEnvironmentVariable('EARCRAWLER_API_HOST', '127.0.0.1', 'Machine')
[System.Environment]::SetEnvironmentVariable('EARCRAWLER_API_PORT', '9001', 'Machine')
[System.Environment]::SetEnvironmentVariable('EARCRAWLER_FUSEKI_URL', 'http://127.0.0.1:3030/ear/query', 'Machine')
[System.Environment]::SetEnvironmentVariable('EARCRAWLER_REMOTE_LLM_POLICY', 'deny', 'Machine')
[System.Environment]::SetEnvironmentVariable('EARCRAWLER_ENABLE_REMOTE_LLM', '0', 'Machine')
```

Only set these when you need them:

```powershell
[System.Environment]::SetEnvironmentVariable('EARCRAWLER_API_KEYS', 'ops=<replace-with-secret>', 'Machine')
[System.Environment]::SetEnvironmentVariable('TRADEGOV_API_KEY', '<replace-with-secret>', 'Machine')
[System.Environment]::SetEnvironmentVariable('GROQ_API_KEY', '<replace-with-secret>', 'Machine')
[System.Environment]::SetEnvironmentVariable('NVIDIA_NIM_API_KEY', '<replace-with-secret>', 'Machine')
```

If you enable optional remote answering, also set:

```powershell
[System.Environment]::SetEnvironmentVariable('EARCRAWLER_REMOTE_LLM_POLICY', 'allow', 'Machine')
[System.Environment]::SetEnvironmentVariable('EARCRAWLER_ENABLE_REMOTE_LLM', '1', 'Machine')
```

### 4. Install the Windows service with NSSM

This guide assumes NSSM is installed at `C:\tools\nssm\nssm.exe`. If it is elsewhere, substitute that path.

```powershell
$nssm = 'C:\tools\nssm\nssm.exe'
$runtimeRoot = 'C:\Program Files\EarCrawler\runtime'
$workspaceRoot = 'C:\ProgramData\EarCrawler\workspace'
$logRoot = 'C:\ProgramData\EarCrawler\logs'
$venvPython = Join-Path $runtimeRoot '.venv\Scripts\python.exe'

& $nssm install EarCrawler-API $venvPython `
  '-m' 'uvicorn' 'service.api_server.server:app' '--host' '127.0.0.1' '--port' '9001'
& $nssm set EarCrawler-API AppDirectory $workspaceRoot
& $nssm set EarCrawler-API AppStdout (Join-Path $logRoot 'api-service.log')
& $nssm set EarCrawler-API AppStderr (Join-Path $logRoot 'api-service.log')
& $nssm set EarCrawler-API Start SERVICE_AUTO_START
```

Configure the service account and recovery policy in NSSM or Services:

- run as the intended least-privilege service account
- set recovery to restart on failure
- keep the service bound to the single host; do not claim multi-instance support

### 5. Start and verify

```powershell
C:\tools\nssm\nssm.exe start EarCrawler-API
Invoke-WebRequest http://127.0.0.1:9001/health -UseBasicParsing
Invoke-WebRequest `
  -Uri http://127.0.0.1:3030/ear/query `
  -Method Post `
  -ContentType 'application/sparql-query' `
  -Body 'SELECT (1 AS ?ok) WHERE { } LIMIT 1' `
  -UseBasicParsing
```

Healthy install criteria:

- the service is running in `Get-Service EarCrawler-API`
- `/health` returns HTTP 200
- the configured Fuseki query endpoint returns HTTP 200 to the trivial `SELECT`

## Upgrade

Use the same dedicated virtual environment. Do not upgrade by copying a source checkout onto the host.

1. Copy the new signed wheel and checksum file to the host.
2. Stop the service.
3. Take a backup snapshot using the procedure below.
4. Reinstall the wheel into the existing virtual environment.
5. Restart the service.
6. Re-run the health checks.

Commands:

```powershell
$venvPython = 'C:\Program Files\EarCrawler\runtime\.venv\Scripts\python.exe'
C:\tools\nssm\nssm.exe stop EarCrawler-API
& $venvPython -m pip install --upgrade --force-reinstall 'C:\path\to\earcrawler-0.2.6-py3-none-any.whl'
C:\tools\nssm\nssm.exe start EarCrawler-API
Invoke-WebRequest http://127.0.0.1:9001/health -UseBasicParsing
```

If the release changes environment variables, update the machine-scoped values before restarting the service.

## Backup

Take backups before every upgrade and before any configuration or secret change.

### What to back up

- `C:\ProgramData\EarCrawler\config`
- `C:\ProgramData\EarCrawler\logs`
- `C:\ProgramData\EarCrawler\workspace`
- `%PROGRAMDATA%\EarCrawler\audit`
- `%PROGRAMDATA%\EarCrawler\spool`
- the currently installed wheel version and release checksum
- the service configuration and current machine environment values

If the Fuseki endpoint is local to the same host, back up its TDB2 data directory and service definition in the same maintenance window. EarCrawler does not automate that step.

### Backup procedure

```powershell
$stamp = Get-Date -Format 'yyyyMMdd-HHmmss'
$backup = "C:\ProgramData\EarCrawler\backups\$stamp"
$venvPython = 'C:\Program Files\EarCrawler\runtime\.venv\Scripts\python.exe'

C:\tools\nssm\nssm.exe stop EarCrawler-API
New-Item -ItemType Directory -Force -Path $backup | Out-Null

Copy-Item 'C:\ProgramData\EarCrawler\config' (Join-Path $backup 'config') -Recurse -Force -ErrorAction SilentlyContinue
Copy-Item 'C:\ProgramData\EarCrawler\logs' (Join-Path $backup 'logs') -Recurse -Force -ErrorAction SilentlyContinue
Copy-Item 'C:\ProgramData\EarCrawler\workspace' (Join-Path $backup 'workspace') -Recurse -Force -ErrorAction SilentlyContinue
Copy-Item "$env:PROGRAMDATA\EarCrawler\audit" (Join-Path $backup 'audit') -Recurse -Force -ErrorAction SilentlyContinue
Copy-Item "$env:PROGRAMDATA\EarCrawler\spool" (Join-Path $backup 'spool') -Recurse -Force -ErrorAction SilentlyContinue

& $venvPython -m pip show earCrawler | Set-Content (Join-Path $backup 'pip-show.txt')
reg export 'HKLM\SYSTEM\CurrentControlSet\Control\Session Manager\Environment' (Join-Path $backup 'machine-env.reg') /y | Out-Null
sc.exe qc EarCrawler-API > (Join-Path $backup 'service-qc.txt')
sc.exe qfailure EarCrawler-API > (Join-Path $backup 'service-failure.txt')
```

After the backup completes, either keep the service stopped for maintenance or start it again:

```powershell
C:\tools\nssm\nssm.exe start EarCrawler-API
```

## Restore

Restore is for rebuilding the same version and state after host loss or operator error.

1. Recreate the standard host layout.
2. Recreate the Python virtual environment.
3. Reinstall the same wheel version captured in the backup.
4. Restore `config`, `logs`, `workspace`, `audit`, and `spool`.
5. Reapply the required machine environment variables from the reviewed backup record.
6. Recreate the NSSM service.
7. Start the service and run the health checks.

If the original host used a local Fuseki instance, restore that dataset before you start EarCrawler.

## Rollback

Rollback is the fast path for a bad release when the prior host state is still available.

1. Stop the service.
2. Reinstall the last known-good wheel.
3. Reapply the prior machine environment values if the failed release changed them.
4. Restore the prior `config` or `workspace` snapshot if the failed release mutated them.
5. Start the service.
6. Verify `/health`.

Commands:

```powershell
$venvPython = 'C:\Program Files\EarCrawler\runtime\.venv\Scripts\python.exe'
C:\tools\nssm\nssm.exe stop EarCrawler-API
& $venvPython -m pip install --upgrade --force-reinstall 'C:\path\to\earcrawler-0.2.5-py3-none-any.whl'
C:\tools\nssm\nssm.exe start EarCrawler-API
Invoke-WebRequest http://127.0.0.1:9001/health -UseBasicParsing
```

Rollback is not complete until:

- the prior wheel version is installed
- the service is running
- `/health` is green
- any local Fuseki dependency is also back on the prior known-good state

## Secret rotation

The authoritative deployment path uses machine-scoped environment variables for service secrets. Rotate secrets there, then restart the affected service or job host.

### Rotate inbound API keys

```powershell
[System.Environment]::SetEnvironmentVariable('EARCRAWLER_API_KEYS', 'ops=<new-secret>', 'Machine')
C:\tools\nssm\nssm.exe restart EarCrawler-API
```

Clients must then send `X-Api-Key: ops:<new-secret>` or the unlabeled raw secret form if you choose not to use labels.

### Rotate upstream or provider secrets

Examples:

```powershell
[System.Environment]::SetEnvironmentVariable('TRADEGOV_API_KEY', '<new-secret>', 'Machine')
[System.Environment]::SetEnvironmentVariable('GROQ_API_KEY', '<new-secret>', 'Machine')
[System.Environment]::SetEnvironmentVariable('NVIDIA_NIM_API_KEY', '<new-secret>', 'Machine')
C:\tools\nssm\nssm.exe restart EarCrawler-API
```

If a secret is no longer needed, remove it explicitly:

```powershell
[System.Environment]::SetEnvironmentVariable('GROQ_API_KEY', $null, 'Machine')
```

### Rotate optional remote-answering posture

To disable remote answering again:

```powershell
[System.Environment]::SetEnvironmentVariable('EARCRAWLER_REMOTE_LLM_POLICY', 'deny', 'Machine')
[System.Environment]::SetEnvironmentVariable('EARCRAWLER_ENABLE_REMOTE_LLM', '0', 'Machine')
C:\tools\nssm\nssm.exe restart EarCrawler-API
```

## Operational notes

- Keep at least the current and prior signed wheels plus their checksum files on the host or in the release vault.
- Record the exact wheel filename and backup snapshot id used for every upgrade and rollback.
- Treat `C:\ProgramData\EarCrawler\config\telemetry.json` as part of the deployment state if telemetry is enabled.
- Do not use this guide to claim container or multi-instance support. The supported target remains one Windows host.
- If scale-out becomes in scope later, start from `docs/ops/multi_instance_deferred.md` instead of inferring support from this guide.
